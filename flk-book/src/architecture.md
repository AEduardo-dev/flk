# Architecture

flk consists of:

- **Binary crate (`src/main.rs`)**: CLI entrypoint using `clap` with subcommands (`init`, `search`, `deep-search`, `add`, `remove`, `env`, `cmd`, `lock`, `update`, `activate`, `export`, `direnv`, `hook`, `completions`).
- **Library crate (`src/lib.rs` and `src/flake/**`)**: flake generation/parsing, profiles, overlays, shell hooks, and nix helpers.
- **Nix integration (`src/nix`)**: wrappers for invoking `nix` commands and handling output.
- **Utilities (`src/utils`)**: backups, spinners/visual output, table/list rendering.

## Data layout
- **flake.nix**: root flake generated by `flk init`.
- **.flk/profiles/*.nix**: per-profile devShell definitions (default profile is imported by `.flk/profiles/default.nix`).
- **.flk/pins.nix**: pinned package metadata when versions are specified.
- **.flk/overlays.nix**: generated overlays used by profiles.
- **.flk/backups/**: lockfile backups created by `flk update`.

## Command flow
- CLI parses args via `clap` in `src/main.rs`.
- Each subcommand calls a `run_*` function in `src/commands/`.
- Flake parsing/mutation uses helpers in `flk::flake::parsers` and `flk::flake::interfaces`.
- Nix operations (search, versions, updates) run through `nix` wrappers with spinners for UX.
